# ReadmeUpdater

## Назначение

Утилита для автоматического обновления README-файлов в проекте HWGA. Сканирует структуру папок с темами и задачами, анализирует прогресс выполнения и генерирует актуальные таблицы прогресса в формате Markdown. Обновляет содержимое README-файлов между специальными маркерами.

## Состав проекта

В директории утилиты находятся:
- **Program.cs** — основной исходный файл с логикой работы (на C#);
- **ReadmeUpdater.csproj** — файл проекта .NET для сборки утилиты;
- **bin, obj** — папки с промежуточными файлами сборки (формируются автоматически);
- вспомогательные файлы (шаблоны, конфигурационные данные).

### Абстракции
- **IFileService.cs**
  - Определяет метод `UpdateReadmeTable`, который обновляет таблицу в README-файле.
- **IMarkdownGenerator.cs**
  - Определяет метод `GenerateTable`, который генерирует Markdown-таблицу на основе прогресса тем.
- **ITaskResolutionStrategy.cs**
  - Определяет метод `IsResolved`, который проверяет, выполнена ли задача по заданной стратегии.
- **ITaskScanner.cs**
  - Определяет метод `ScanThemes`, который сканирует и возвращает прогресс всех тем.

### Сервисы
- **FileService.cs**
  - Реализует интерфейс `IFileService` для обновления таблицы в README-файле.
- **MarkdownGenerator.cs**
  - Реализует интерфейс `IMarkdownGenerator` для генерации Markdown-таблицы.
- **TaskScanner.cs**
  - Реализует интерфейс `ITaskScanner` для сканирования тем и возврата прогресса.

### Стратегии
- **AllRequirementsStrategy.cs**
  - Реализует стратегию, требующую выполнения всех подзадач.
- **AnyRequirementStrategy.cs**
  - Реализует стратегию, требующую выполнения хотя бы одной подзадачи.
- **FileExistenceStrategy.cs**
  - Реализует стратегию, проверяющую наличие файлов с определенным расширением.
- **TestExistenceStrategy.cs**
  - Реализует стратегию, проверяющую наличие тестовых файлов.

## Как работает

1. **Сканирование структуры:** утилита обходит директорию `src/app` и находит все темы (папки вида `Theme01.CLR`, `Theme00.Initial` и т. д.).
2. **Анализ задач:** для каждой темы проверяет подпапки `Easy`, `Medium`, `Hard`.
3. **Определение статуса:** задача считается выполненной, если в её папке есть файл с расширением `.cs` или соответствующий критерий из стратегии.
4. **Формирование таблицы:** создаёт Markdown-таблицу с колонками:
   - название темы;
   - прогресс по уровням сложности (`Easy/Medium/Hard` — в формате `X/Y`, где X — выполнено, Y — всего);
   - общий статус темы (✅ — все задачи выполнены, 🟡 — есть нерешённые);
   - общее количество задач и выполненное количество задач.

## Как запустить

1. Убедитесь, что у вас установлена .NET SDK (версия 10.0 или выше).
2. Откройте терминал в директории с утилитой.
3. Выполните команду:
   ```bash
   dotnet run
   ```
4. Утилита автоматически обновит все README-файлы с актуальными данными.

## Параметры запуска (опционально)

Можно передать дополнительные аргументы:
- `--root-path <путь>` — указать корневую директорию проекта (по умолчанию ищет автоматически);
- `--dry-run` — режим «прогона»: утилита покажет, какие изменения будут внесены, но не сохранит их;
- `--verbose` — подробный вывод отладочной информации.

Пример:
```bash
dotnet run --root-path ./hwga-project --verbose
```

## Формат таблицы прогресса

Пример генерируемой таблицы:
```markdown
<!-- PROGRESS_TABLE_START -->
| Тема | Easy | Medium | Hard | Status |
| :--- | :---: | :---: | :---: | :--- |
| Theme00.Initial | 0/0 | 0/0 | 1/1 | ✅ 1/1 |
| Theme01.CLR | 1/1 | 0/2 | 0/2 | 🟡 1/5 |
<!-- PROGRESS_TABLE_END -->
```

## Требования

- .NET SDK (10.0+).
- Структура проекта с директорией `src/app` и темами в формате `ThemeXX.Название`.
- README-файлы с маркерами `<!-- PROGRESS_TABLE_START -->` и `<!-- PROGRESS_TABLE_END -->`.

## Ограничения

- Работает только с C#-файлами (признак выполненной задачи — наличие `.cs` в папке задачи) или соответствующими стратегиями.
- Зависит от строгой структуры папок (темы → уровни сложности → задачи).
- Не поддерживает ручную корректировку таблицы между маркерами (изменения будут перезаписаны).

## Развитие и доработки

Если вам нужно добавить новые функции (например, поддержку других языков, изменение формата таблицы), вносите правки в `Program.cs` и обновляйте `ReadmeUpdater.csproj`. После изменений пересоберите утилиту командой `dotnet build`.
