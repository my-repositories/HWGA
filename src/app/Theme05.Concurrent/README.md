# Практические задачи по теме «Concurrent» (.NET / C")


Сборник практических заданий для углублённого изучения конкурентного программирования в экосистеме .NET. Освоение темы позволит создавать потокобезопасные приложения, предотвращать гонки данных и дедлоки, а также эффективно работать с общими ресурсами в многопоточной среде.

## 🎯 Обзор темы


Конкурентное программирование (Concurrency) — подход к разработке, обеспечивающий безопасное выполнение кода в многопоточной среде при одновременном доступе нескольких потоков к общим ресурсам. Ключевая цель — гарантировать целостность данных и корректность работы приложения при параллельном выполнении операций.


Ключевые механизмы и компоненты:


- **`ConcurrentDictionary<TKey, TValue>`** — потокобезопасная хэш‑таблица: позволяет нескольким потокам одновременно читать и записывать данные без явных блокировок, используя технику striped locking.
- **`ConcurrentQueue<T>`** — потокобезопасная очередь FIFO: обеспечивает безопасный доступ из нескольких потоков, подходит для сценариев producer‑consumer.
- **`ConcurrentBag<T>`** — неупорядоченная коллекция для быстрого добавления и извлечения элементов: оптимизирована для случаев, когда один поток добавляет элементы, а другой их обрабатывает.
- **Примитивы синхронизации (`lock`, `Mutex`, `SemaphoreSlim`, `ReaderWriterLockSlim`)** — инструменты для защиты критических секций кода: предотвращают гонки данных, контролируют доступ к общим ресурсам.
- **`BlockingCollection<T>`** — потокобезопасная коллекция с поддержкой блокировки: автоматически приостанавливает операции чтения при отсутствии элементов и записи при достижении лимита.
- **`CountdownEvent`** — примитив синхронизации для ожидания завершения группы операций: позволяет потоку блокироваться до тех пор, пока счётчик не достигнет нуля.


## ✅ Что вы освоите

- Научитесь использовать потокобезопасные коллекции (`ConcurrentDictionary`, `ConcurrentQueue`, `ConcurrentBag`) для работы с общими данными из нескольких потоков.
- Освоите применение примитивов синхронизации (`lock`, `SemaphoreSlim`, `ReaderWriterLockSlim`) для защиты критических секций и предотвращения гонок данных.
- Поймёте, как организовать взаимодействие потоков через `BlockingCollection<T>` в сценариях producer‑consumer с автоматическим управлением блокировками.
- Научитесь координировать завершение параллельных операций с помощью `CountdownEvent` и `Barrier`, обеспечивая корректную синхронизацию потоков.
- Освоите диагностику и устранение типичных проблем конкурентности (дедлоков, гонок данных) с помощью профилировщиков и отладочных инструментов.

## 📂 Структура раздела


Задачи организованы по уровням сложности для постепенного погружения в тему:


- **Easy** (`HWGA.Concurrent.Easy`) — базовые упражнения на использование потокобезопасных коллекций: добавление и чтение элементов из `ConcurrentDictionary`, работа с `ConcurrentQueue`. Цель: закрепить понимание принципов потокобезопасности, научиться выбирать подходящую коллекцию для конкретного сценария.
- **Medium** (`HWGA.Concurrent.Medium`) — задачи на комбинирование примитивов синхронизации и коллекций: реализация producer‑consumer очередей через `BlockingCollection`, использование `SemaphoreSlim` для ограничения параллелизма. Цель: научиться координировать потоки, обеспечивать безопасный доступ к ресурсам при высокой нагрузке.
- **Hard** (`HWGA.Concurrent.Hard`) — комплексные сценарии: проектирование кэша с истечением срока действия (TTL) на базе `ConcurrentDictionary`, обработка дедлоков и гонок данных в распределённых системах. Цель: отработать навыки проектирования масштабируемых конкурентных решений, диагностики и оптимизации производительности.


## 🛠 Методика работы


1. Перед реализацией конкурентного кода моделируйте сценарии одновременного доступа: продумайте, какие потоки будут читать/записывать данные, какие ресурсы являются общими.
2. При работе с примитивами синхронизации минимизируйте время удержания блокировок: выполняйте тяжёлые операции вне критических секций, используйте `SemaphoreSlim` или `ReaderWriterLockSlim` вместо `lock` для сложных сценариев.
3. Для диагностики проблем конкурентности применяйте инструменты Visual Studio: «Параллельный стек вызовов» и «Профилировщик конкуренции» — они помогут выявить дедлоки и избыточные блокировки.
4. После решения анализируйте использование памяти через PerfView: проверьте, нет ли избыточных аллокаций (например, при частом вызове `Keys`/`Values` у `ConcurrentDictionary`).

## 💡 Дополнительный совет


Для проверки решений задач уровня **Hard** используйте `PerfView` для сбора трассировки CPU и памяти во время выполнения. Сравните время обработки с наивной реализацией на базе `Dictionary` + `lock`: убедитесь, что конкурентное решение демонстрирует выигрыш в производительности при высокой параллельной нагрузке (график CPU Usage должен быть близок к 100 % для всех ядер). Дополнительно проверьте отсутствие дедлоков через «Параллельный стек вызовов» в Visual Studio — это подтвердит корректность синхронизации и эффективность конкурентного подхода.
