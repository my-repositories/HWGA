## HelloWorld (Hard)
**Задача:**

Создать расширяемую архитектуру для вывода сообщения «Hello, World!» на C#, воспроизводящую логику и структуру предоставленного кода с возможностью дальнейшего расширения. Система должна демонстрировать применение паттернов проектирования (Factory, Dependency Injection) и обеспечивать слабую связность компонентов через интерфейсы. Использовать заданную структуру пространств имён: `HWGA`, `HWGA.Interfaces` и `HWGA.Impl`.

**Примеры:**

1. Вход: запуск приложения без дополнительных параметров  
Выход: `Hello, World!` (в консоли)  
Объяснение: приложение использует фабрики для создания всех компонентов, получает строку «Hello, World!» и выводит её в консоль через стандартную стратегию печати.

2. Вход: искусственно смоделированная ошибка при настройке стратегии печати (например, недоступность потока вывода)  
Выход: исключение в консоли:  
```
Unhandled exception. System.Exception: Failed to create IPrintStrategy: -1
```  
Объяснение: система корректно обрабатывает ошибку инициализации `IPrintStrategy`, возвращает код ошибки `-1` через `IStatusCode` и выбрасывает исключение с описанием проблемы.


3. Вход: ошибка при выполнении операции печати (например, проблемы с доступом к потоку вывода)  
Выход: исключение в консоли:  
```
Unhandled exception. System.Exception: Failed to print: -1
```  
Объяснение: если операция печати завершается с ошибкой, система возвращает код `-1`, что приводит к выбросу исключения в основном классе `HelloWorld`.

**Ограничения:**

* строго придерживаться заданной архитектуры с интерфейсами `IHelloWorld`, `IHelloWorldString`, `IPrintStrategy`, `IStatusCode`;
* разместить интерфейсы в пространстве имён `HWGA.Interfaces`;
* разместить реализации классов в пространстве имён `HWGA.Impl`;
* основной класс `HelloWorld` разместить в пространстве имён `HWGA`;
* использовать фабрики (`HelloWorldFactory`, `StringFactory`, `PrintStrategyFactory`) для создания всех объектов;
* обеспечить обработку ошибок на каждом этапе (создание объектов, настройка печати, вывод) через `IStatusCode`;
* не модифицировать сигнатуры методов в интерфейсах;
* код должен компилироваться и запускаться в среде .NET 6.0+.

**Подсказка:** 

обратите внимание на цепочку вызовов: `HelloWorld` → `HelloWorldFactory` → `HelloWorldImplementation` → другие фабрики. Убедитесь, что все зависимости внедряются через фабрики, а не создаются напрямую. Используйте статические свойства `Instance` для реализации паттерна Singleton в фабриках. Пространство имён для интерфейсов: `namespace HWGA.Interfaces;`, для реализаций: `namespace HWGA.Impl;`, для главного класса: `namespace HWGA;`.

**Что проверить в решении:**

* корректность работы всей цепочки фабрик и интерфейсов (создание объектов через фабрики, передача зависимостей);
* точный вывод строки «Hello, World!» в стандартный поток вывода при успешном выполнении;
* правильная обработка ошибок на этапах настройки печати и вывода (возвращение кодов `-1` через `IStatusCode` и выброс исключений);
* соблюдение принципов слабой связности (компоненты взаимодействуют только через интерфейсы);
* возможность расширения системы без изменения существующего кода (например, добавление новой реализации `IPrintStrategy` для вывода в файл);
* отсутствие прямых зависимостей между классами реализации (все связи через интерфейсы и фабрики);
* соответствие структуры проекта описанной в техническом задании (наличие всех классов и интерфейсов, корректное размещение по пространствам имён);
* корректная работа с `System.Console.Out` вместо Java‑специфичных классов.
