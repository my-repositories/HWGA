# Практические задачи по теме «Multithreading» (.NET / C")

Сборник практических заданий для углублённого изучения многопоточного программирования в экосистеме .NET. Освоение темы позволит создавать отзывчивые приложения, эффективно использовать ресурсы процессора и решать задачи параллельной обработки данных.

## 🎯 Обзор темы

Многопоточность (Multithreading) — механизм выполнения нескольких потоков кода одновременно в рамках одного процесса. Позволяет повысить производительность приложений и обеспечить их отзывчивость, особенно в сценариях с длительными операциями (сетевые запросы, обработка файлов и т. д.).


Ключевые механизмы и компоненты:

- **`Thread`** — базовый класс для создания и управления потоками выполнения: даёт полный контроль над потоком, но требует ручной настройки и управления.
- **`ThreadPool`** — пул потоков для эффективного переиспользования: снижает накладные расходы на создание потоков, подходит для коротких задач.
- **`Task` и `Task<T>`** — абстракция над потоками для асинхронных и параллельных операций: упрощает работу с многопоточностью через TPL (Task Parallel Library).
- **`CancellationToken`** — механизм отмены длительных операций: позволяет корректно останавливать потоки и задачи по запросу, избегая «зависших» процессов.
- **Примитивы синхронизации (`lock`, `Mutex`, `SemaphoreSlim`, `Monitor`)** — инструменты для защиты общих ресурсов: предотвращают гонки данных (race conditions) и обеспечивают целостность данных при доступе из нескольких потоков.
- **`async`/`await`** — синтаксическая конструкция для асинхронного программирования: упрощает написание неблокирующего кода, часто используется вместе с многопоточностью.

## ✅ Что вы освоите

- Научитесь создавать и управлять потоками через класс `Thread`, понимать их жизненный цикл и особенности работы.
- Освоите использование `ThreadPool` для оптимизации выполнения коротких фоновых задач и снижения накладных расходов.
- Поймёте, как применять `Task` и `Task<T>` для организации параллельного выполнения операций с контролем результатов.
- Научитесь безопасно работать с общими данными через примитивы синхронизации (`lock`, `SemaphoreSlim`), предотвращая гонки данных и дедлоки.
- Освоите отмену длительных операций с помощью `CancellationToken`, обеспечивая корректное завершение потоков и задач.

## 📂 Структура раздела

Задачи организованы по уровням сложности для постепенного погружения в тему:

- **Easy** (`HWGA.Multithreading.Easy`) — базовые упражнения на создание потоков через `Thread`, запуск простых фоновых операций, использование `ThreadPool`. Цель: закрепить понимание жизненного цикла потока, научиться запускать и останавливать потоки, освоить базовые сценарии фоновой обработки.
- **Medium** (`HWGA.Multithreading.Medium`) — задачи на работу с `Task`, параллельное выполнение операций, применение примитивов синхронизации (`lock`). Цель: научиться координировать несколько потоков, обеспечивать потокобезопасность при работе с общими ресурсами, обрабатывать исключения в многопоточном коде.
- **Hard** (`HWGA.Multithreading.Hard`) — комплексные сценарии: реализация producer‑consumer очередей через `BlockingCollection`, использование `SemaphoreSlim` для ограничения параллелизма, обработка дедлоков и гонок данных. Цель: отработать навыки проектирования масштабируемых многопоточных решений, диагностики и устранения типичных проблем параллелизма.

## 🛠 Методика работы

1. Перед запуском многопоточного кода измеряйте время выполнения последовательной версии через `Stopwatch`: это поможет оценить реальный выигрыш от распараллеливания и выявить неоптимальные участки.
2. При работе с общими ресурсами всегда используйте примитивы синхронизации (`lock`, `SemaphoreSlim`) или потокобезопасные коллекции (`ConcurrentDictionary`), чтобы избежать гонок данных.
3. Для диагностики проблем параллелизма применяйте инструменты Visual Studio: «Параллельный стек вызовов» и «Профилировщик конкуренции» — они помогут выявить дедлоки, избыточные блокировки и узкие места.
4. После решения анализируйте использование CPU через Диспетчер задач или PerfView: если нагрузка не достигает 100 % на всех ядрах, ищите неоптимальное распределение задач между потоками или избыточные блокировки.

- *C# 9.0. Полное руководство* (Г. Шилдт) — раздел о многопоточности: разбор классов `Thread`, `ThreadPool`, примеров использования `Task`.
- *Concurrency in C# Cookbook* (С. Клири) — практические рецепты по многопоточности: работа с `CancellationToken`, реализация producer‑consumer, обработка дедлоков.

**Официальные источники:**
- Документация Microsoft по многопоточности в .NET: https://learn.microsoft.com/en-us/dotnet/standard/threading/

**Инструменты:**

- `Visual Studio Diagnostic Tools` — встроенные средства отладки: мониторинг потоков, анализ параллельных стеков, профилирование конкуренции.

## 💡 Дополнительный совет


Для проверки решений задач уровня **Hard** используйте `PerfView` для сбора трассировки CPU и блокировки потоков во время выполнения. Сравните время обработки с последовательной версией и убедитесь, что нагрузка равномерно распределена между ядрами (график CPU Usage должен быть близок к 100 % для всех ядер). Дополнительно проверьте наличие дедлоков через «Параллельный стек вызовов» в Visual Studio — это подтвердит корректность синхронизации и эффективность многопоточного решения.
