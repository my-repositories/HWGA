## Практические задачи по теме «CLR» (.NET / C#)
Сборник практических заданий для углублённого изучения ключевых механизмов Common Language Runtime (CLR) в экосистеме .NET. Упражнения помогут на практике разобраться в работе среды выполнения и закрепить понимание внутренних процессов платформы.
## 🎯 Обзор темы
**Common Language Runtime (CLR)** — это виртуальная машина, управляющая выполнением кода в .NET и обеспечивающая базовые сервисы для приложений. Изучение CLR необходимо для понимания того, как исходный код превращается в машинные инструкции и как оптимизируется работа с системными ресурсами.

* Управление памятью: автоматическое выделение и освобождение памяти через Garbage Collector (GC).
* Система типов (CTS): унифицированный стандарт типов данных, обеспечивающий взаимодействие между различными языками .NET.
* JIT-компиляция: процесс преобразования байт-кода (IL) в оптимизированные машинные команды непосредственно во время выполнения.
* Обработка исключений: интегрированная модель обработки ошибок, обеспечивающая безопасность и стабильность приложений.
* Загрузка сборок: поиск, верификация и загрузка бинарных файлов (Assembly) в память процесса.

## ✅ Что вы освоите

* Поймёте разницу в хранении и поведении типов значений (struct) и ссылочных типов (class) в памяти.
* Освоите низкоуровневую работу с метаданными типов через дескрипторы RuntimeTypeHandle.
* Научитесь динамически загружать и изолировать код, используя механизмы AssemblyLoadContext.
* Поймёте жизненный цикл объектов и влияние различных структур данных на работу Garbage Collector.
* Оптимизируете производительность алгоритмов, учитывая особенности JIT-компиляции и упаковки (boxing/unboxing).

## 📂 Структура раздела

* Easy (HWGA.CLR.Easy): задания на базовые концепции среды выполнения. Основная цель — закрепить понимание разницы между типами данных и принципов их передачи в методы.
* Medium (HWGA.CLR.Medium): упражнения на использование рефлексии и работу с дескрипторами типов рантайма. Цель заключается в изучении способов анализа метаданных объектов без прямого использования высокоуровневых методов.
* Hard (HWGA.CLR.Hard): комплексные задачи на изоляцию кода и управление жизненным циклом сборок. Пользователю предстоит реализовать сценарии динамической загрузки и полной выгрузки модулей из памяти.

## 🛠 Методика работы

* Анализируйте IL-код: используйте инструменты вроде ildasm или sharplab.io, чтобы увидеть, как компилятор разворачивает ваш C# код в инструкции промежуточного языка.
* Профилируйте аллокации: при выполнении задач отслеживайте количество создаваемых объектов, чтобы понимать, как ваш код нагружает кучу (Heap).
* Экспериментируйте с типами: меняйте class на struct в задачах и наблюдайте за изменением поведения программы при копировании и сравнении объектов.
* Используйте отладчик: изучайте окно «Locals» и адреса объектов в памяти, чтобы визуализировать процесс выделения ресурсов в CLR.

## 📚 Рекомендуемые ресурсы

* Джеффри Рихтер — «CLR via C#»: фундаментальный труд, описывающий устройство рантайма от загрузки сборок до работы сборщика мусора.
* Конрад Кокоса — «Pro .NET Memory Management»: глубокое руководство по управлению памятью, незаменимое для понимания внутреннего устройства GC.
* Официальная документация Microsoft: Common Language Runtime (CLR) overview — актуальные сведения об архитектуре среды.
* Инструменты диагностики: используйте dotMemory для анализа утечек памяти и BenchmarkDotNet для точного измерения производительности участков кода.

**Совет:** Для задач уровня Hard проверяйте успешную выгрузку сборки с помощью WeakReference: после вызова метода выгрузки и принудительного GC.Collect(), слабая ссылка должна указывать на null, подтверждая отсутствие удерживающих связей в памяти.
