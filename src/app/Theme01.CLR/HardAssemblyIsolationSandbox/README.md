## AssemblyIsolationSandbox (Hard)
Задача: Реализовать логику безопасного выполнения внешнего кода. Программа должна загружать байты сборки (Assembly), находить в ней все типы, реализующие интерфейс IRunnable, и запускать их в контексте, который позволяет выгрузить эти данные из памяти после завершения работы.
Пример:
Вход: Путь к внешней .dll файлу.
Выход: Результат выполнения метода Run() из загруженной сборки.
Объяснение: Сборка загружается динамически, выполняется, и после этого GC может очистить память от этой сборки.
Ограничения:

* Для .NET Core/5+ использовать AssemblyLoadContext с поддержкой isCollectible: true;
* Обеспечить отсутствие утечек памяти (проверить WeakReference на сборку).

Подсказка: Чтобы сборка выгрузилась, необходимо разорвать все ссылки на типы из этой сборки. Шаблон: namespace HWGA.CLR.Hard.AssemblyIsolationSandbox;
Что проверить в решении:

* Использование AssemblyLoadContext для изоляции;
* Корректный поиск типов через рефлексию;
* Реализация механизма выгрузки (Unload) и подтверждение очистки памяти.

Решение: Смотрите Program.cs.
