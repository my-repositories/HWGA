## ObjectLifetimeAndGC (Medium)
**Задача:** Реализовать класс, отслеживающий своё время жизни в CLR, с демонстрацией работы сборщика мусора (GC). Класс должен регистрировать создание, финализацию и возможность принудительной очистки.

**Пример:**

Вход 1:
```csharp
using (var tracker = new LifecycleTracker("Tracker1"))
{
    // работа с объектом
}
// вызов GC.Collect()
```

Выход 1:
```
Tracker1 created
Tracker1 disposed
Tracker1 finalized
```

Вход 2:
```csharp
var tracker = new LifecycleTracker("Tracker2");
tracker = null;
GC.Collect();
```

Выход 2:
```
Tracker2 created
Tracker2 finalized
```

Объяснение: В первом случае объект явно освобождается через `using`, во втором — сборщик мусора находит недостижимый объект и вызывает финализатор.

**Подсказка:** Используйте шаблон `IDisposable` для корректного освобождения ресурсов и переопределите метод `Finalize` (или используйте деструктор). Пространство имён: `namespace HWGA.CLR.Medium.ObjectLifetimeAndGC;`.

**Ограничения:**
* Класс должен реализовывать `IDisposable`.
* Финализатор должен выводить сообщение о финализации.
* Не должно быть утечек памяти (все неуправляемые ресурсы освобождены).
* Код должен работать в .NET 6+ и демонстрировать разницу между `Dispose` и финализацией.

**Что проверить в решении:**
* Корректность сообщений о создании, освобождении и финализации.
* Обработка случая, когда `Dispose` не вызван явно.
* Отсутствие утечек памяти при многократном создании объектов.

**Решение:** Смотрите `Program.cs`.
