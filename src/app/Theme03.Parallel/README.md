# Практические задачи по теме «Parallel» (.NET / C")


Сборник практических заданий для углублённого изучения параллельного программирования в экосистеме .NET с использованием Task Parallel Library (TPL). Освоение темы позволит эффективно использовать многоядерные процессоры, ускорять выполнение вычислительно сложных задач и создавать высокопроизводительные приложения.


## 🎯 Обзор темы

Параллельное программирование в .NET — подход, позволяющий выполнять несколько операций одновременно на разных ядрах процессора, что существенно ускоряет обработку данных при наличии независимых вычислений.


Ключевые механизмы и компоненты:

- **`Parallel.For`/`Parallel.ForEach`** — методы для параллельного выполнения итераций цикла: автоматически распределяют работу между потоками пула, оптимизируя нагрузку.
- **`PLINQ` (Parallel LINQ)** — расширение LINQ для параллельной обработки коллекций: позволяет распараллеливать запросы через `AsParallel()`, автоматически объединяя результаты.
- **`Task.Parallel.Invoke`** — метод для параллельного запуска набора действий: выполняет переданный массив делегатов параллельно, ожидая завершения всех.
- **`ParallelOptions`** — настройки для управления параллельным выполнением: позволяют ограничить степень параллелизма, задать `CancellationToken` или пользовательский планировщик задач.
- **`ConcurrentDictionary`/`ConcurrentBag`** — потокобезопасные коллекции: обеспечивают безопасный доступ из нескольких потоков без явных блокировок, снижая риск дедлоков.
- **`Partitioner`** — инструмент для оптимизации распределения данных: позволяет настраивать секционирование коллекций в `Parallel.ForEach` и PLINQ для балансировки нагрузки между потоками.

## ✅ Что вы освоите

- Научитесь применять `Parallel.For` и `Parallel.ForEach` для ускорения обработки больших наборов данных.
- Освоите использование PLINQ (`AsParallel()`) для параллельных запросов к коллекциям с автоматическим объединением результатов.
- Поймёте, как управлять степенью параллелизма через `ParallelOptions` и ограничивать число потоков для предотвращения перегрузки системы.
- Научитесь использовать потокобезопасные коллекции (`ConcurrentDictionary`, `ConcurrentBag`) для агрегации данных из параллельных операций.
- Освоите диагностику и оптимизацию параллельного кода с помощью профилировщиков и логирования.

## 📂 Структура раздела

Задачи организованы по уровням сложности для постепенного погружения в тему:


- **Easy** (`HWGA.Parallel.Easy`) — базовые упражнения на применение `Parallel.For` и `Parallel.ForEach`: обработка массивов, подсчёт сумм, фильтрация данных. Цель: закрепить синтаксис параллельных циклов, понять принцип распределения итераций между потоками.
- **Medium** (`HWGA.Parallel.Medium`) — задачи на комбинирование PLINQ и `ParallelOptions`: параллельная обработка файлов, агрегация результатов через `ConcurrentDictionary`, настройка лимита потоков. Цель: научиться координировать несколько параллельных процессов, обеспечивать потокобезопасность.
- **Hard** (`HWGA.Parallel.Hard`) — комплексные сценарии: реализация конвейеров обработки данных с `TPL Dataflow`, оптимизация через кастомные `Partitioner`, обработка исключений в параллельных операциях. Цель: отработать навыки проектирования масштабируемых параллельных решений для реальных нагрузок.

## 🛠 Методика работы

1. Перед запуском параллельного кода измеряйте время выполнения последовательной версии через `Stopwatch`: это позволит оценить реальный выигрыш от распараллеливания.
2. При работе с общими ресурсами (коллекции, файлы) используйте потокобезопасные структуры (`ConcurrentBag`, `BlockingCollection`) или явные блокировки (`lock`), чтобы избежать гонок данных.
3. Для диагностики проблем параллелизма применяйте инструменты Visual Studio: «Параллельный стек вызовов» и «Профилировщик конкуренции» — они помогут выявить дедлоки и узкие места.
4. После решения анализируйте использование CPU через Диспетчер задач или PerfView: если нагрузка не достигает 100 % на всех ядрах, ищите неоптимальное секционирование или избыточные блокировки.

## 💡 Дополнительный совет


Для проверки решений задач уровня **Hard** используйте `PerfView` для сбора трассировки CPU во время выполнения. Сравните время обработки с последовательной версией и убедитесь, что нагрузка равномерно распределена между ядрами (график CPU Usage должен быть близок к 100 % для всех ядер). Это подтвердит эффективность параллелизации и отсутствие узких мест.
