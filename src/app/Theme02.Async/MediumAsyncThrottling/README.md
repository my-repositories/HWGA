## AsyncProgressWithCancellation (Hard)
Задача: Реализовать метод для обработки большого массива данных (например, парсинг логов), который поддерживает отмену операции через CancellationToken и уведомляет вызывающий код о проценте прогресса через IProgress<T>.
Пример:
Вход: dataSize = 1000, cancellationAfter = 500ms
Выход: Progress: 10%... 20%... Operation cancelled.
Объяснение: Метод успевает обработать часть данных и отправить уведомления, прежде чем токен отмены перейдет в состояние IsCancellationRequested.
Ограничения:

* Проверка токена отмены должна происходить на каждой итерации или через token.ThrowIfCancellationRequested();
* Обновление прогресса не должно происходить слишком часто (например, не чаще чем раз в 1% или раз в 100мс), чтобы не перегружать UI/лог.

Подсказка: Для имитации долгой работы используйте небольшие паузы в цикле. Передавайте CancellationToken во все вложенные асинхронные вызовы. Пространство имён: namespace HWGA.Асинхронность.Hard.AsyncProgressWithCancellation;.
Что проверить в решении:

* Мгновенную реакцию на запрос отмены (без подвисаний);
* Корректную передачу и отображение процентов прогресса;
* Генерацию и обработку OperationCanceledException.

Решение: Смотрите Program.cs.
