## ParallelFileProcessor (Medium)

**Задача:** Написать асинхронный метод для параллельной обработки нескольких текстовых файлов: чтения, подсчёта слов в каждом и суммирования общего количества слов.

**Пример:**

Вход: `["file1.txt", "file2.txt"]` (файлы содержат 150 и 200 слов соответственно)  
Выход: `350`  
Объяснение: Асинхронно считываются оба файла, параллельно считаются слова в каждом, результаты суммируются.

Вход: `["empty.txt", "short.txt"]` (пустой файл и файл с 50 словами)  
Выход: `50`  
Объяснение: Пустой файл даёт 0 слов, короткий — 50, итого 50.

Вход: `[]` (пустой список)  
Выход: `0`  
Объяснение: При отсутствии файлов сумма слов равна нулю.

**Ограничения:**
* Использовать `File.ReadAllTextAsync` для асинхронного чтения файлов.
* Применять `Task.WhenAll` для параллельного выполнения операций чтения и обработки.
* Метод должен иметь сигнатуру `public async Task<int> CountWordsInFilesAsync(string[] filePaths)`.
* Обрабатывать случаи отсутствия файлов (возвращать 0 для отсутствующих).

**Подсказка:** Используйте LINQ для подсчёта слов в строке (разделение по пробелам и фильтрация непустых элементов). Пространство имён: `namespace HWGA.Async.Medium.ParallelFileProcessor;`.

**Что проверить в решении:**
* Корректность подсчёта общего количества слов при наличии нескольких файлов.
* Обработка пустых файлов и пустого списка путей.
* Устойчивость к отсутствию файлов (не должно вызывать исключений).
* Параллельное выполнение операций чтения для ускорения обработки.

**Решение:** Смотрите `Program.cs`.
