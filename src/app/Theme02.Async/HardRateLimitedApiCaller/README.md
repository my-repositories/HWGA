## RateLimitedApiCaller (Hard)

**Задача:** Разработать асинхронный сервис для вызова внешнего API с ограничением частоты запросов (не более N запросов в секунду) и обработкой тайм‑аутов.

**Пример:**

Вход: 10 запросов к API, лимит — 3 запроса/сек  
Выход: Все запросы выполнены последовательно с интервалами ~333 мс между группами по 3, без превышения лимита  
Объяснение: Запросы группируются и отправляются с задержкой, чтобы не превысить лимит 3 запроса/сек.

Вход: Запрос с тайм‑аутом 100 мс к медленному API (ответ через 500 мс)  
Выход: Исключение `TimeoutException` через 100 мс  
Объяснение: Запрос прерывается по тайм‑ауту, не дожидаясь полного ответа.

Вход: Серия из 5 запросов, один из которых возвращает ошибку 503  
Выход: Успешное выполнение остальных 4 запросов, логирование ошибки для 503  
Объяснение: Ошибки отдельных запросов не прерывают выполнение всей последовательности.

**Ограничения:**
* Реализовать ограничение частоты через `SemaphoreSlim` или `System.Threading.Timer`.
* Каждый запрос должен иметь индивидуальный тайм‑аут (настраиваемый параметр).
* Метод должен поддерживать отмену через `CancellationToken`.
* Сигнатура: `public async Task<List<string>> CallApiWithRateLimitAsync(List<string> urls, int maxRequestsPerSecond, TimeSpan timeout, CancellationToken cancellationToken)`.

**Подсказка:** Используйте `Stopwatch` для отслеживания временных интервалов между группами запросов. Для тайм‑аутов применяйте `CancellationTokenSource` с задержкой. Пространство имён: `namespace HWGA.Async.Hard.RateLimitedApiCaller;`.

**Что проверить в решении:**
* Соблюдение лимита запросов в секунду (не более N параллельно).
* Прерывание запросов по тайм‑ауту без ожидания полного ответа.
* Возможность отмены операции через `CancellationToken`.
* Обработка HTTP‑ошибок (5xx, 4xx) без прерывания всей последовательности запросов.
* Логирование ошибок и успешное выполнение оставшихся запросов при частичных сбоях.

**Решение:** Смотрите `Program.cs`.
