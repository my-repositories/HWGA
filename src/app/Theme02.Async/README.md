# Практические задачи по теме «Async» (.NET / C")

Сборник практических заданий для углублённого изучения асинхронного программирования в экосистеме .NET с использованием ключевых слов `async` и `await`. Освоение темы позволит создавать более отзывчивые и масштабируемые приложения за счёт эффективного использования потоков при выполнении длительных операций (сетевые запросы, работа с файлами, обращения к БД и т. д.).

## 🎯 Обзор темы

Асинхронное программирование в C# — подход, позволяющий выполнять длительные операции без блокировки основного потока, что критично для отзывчивости UI‑приложений и пропускной способности серверных решений.

Ключевые механизмы и компоненты:


- **`async`/`await`** — синтаксическая конструкция для упрощения написания асинхронного кода: `async` маркирует метод как асинхронный, `await` приостанавливает его выполнение до завершения задачи без блокировки потока.
- **`Task`/`Task<T>`** — типы, представляющие асинхронные операции; возвращают результат (`Task<T>`) или просто сигнализируют о завершении (`Task`).
- **`CancellationToken`** — механизм отмены длительных операций, позволяющий корректно останавливать асинхронные задачи по запросу.
- **`Task.WhenAll`/`Task.WhenAny`** — методы для управления параллельным выполнением нескольких задач: `WhenAll` ждёт завершения всех, `WhenAny` — первой завершённой.
- **`ConfigureAwait(false)`** — оптимизация, отключающая захват контекста синхронизации; снижает риск дедлоков и повышает производительность в библиотечном коде.
- **`SemaphoreSlim`** — инструмент ограничения параллелизма: контролирует число одновременно выполняемых асинхронных операций для предотвращения перегрузки ресурсов.

## ✅ Что вы освоите

- Научитесь создавать и вызывать асинхронные методы с модификатором `async` и оператором `await`.
- Освоите обработку исключений в асинхронном коде и поймёте особенности распространения ошибок через `Task`.
- Научитесь управлять параллельным выполнением задач с помощью `Task.WhenAll` и `Task.WhenAny`.
- Освоите использование `CancellationToken` для корректной отмены длительных операций.
- Поймёте, как оптимизировать производительность с помощью `ConfigureAwait(false)` и ограничивать параллелизм через `SemaphoreSlim`.

## 📂 Структура раздела

Задачи организованы по уровням сложности для постепенного погружения в тему:

- **Easy** (`HWGA.Async.Easy`) — базовые упражнения на создание и вызов асинхронных методов, работу с `Task.Delay`, имитацию сетевых запросов. Цель: закрепить синтаксис `async`/`await`, понять жизненный цикл `Task`.
- **Medium** (`HWGA.Async.Medium`) — задачи на параллельное выполнение операций, обработку исключений, отмену задач через `CancellationToken`. Цель: научиться координировать несколько асинхронных процессов, обеспечивать устойчивость к ошибкам.
- **Hard** (`HWGA.Async.Hard`) — комплексные сценарии: ограничение параллелизма (`SemaphoreSlim`), оптимизация контекста (`ConfigureAwait`), работа с потоковыми асинхронными операциями. Цель: отработать навыки проектирования масштабируемых асинхронных решений для реальных нагрузок.

## 🛠 Методика работы

1. Перед решением задачи анализируйте, какой тип `Task` подходит для сценария (например, `Task<string>` для возврата данных, `Task` для операций без результата).
2. При отладке асинхронного кода используйте `Stopwatch` или `BenchmarkDotNet` для измерения времени выполнения и выявления узких мест.
3. После решения проверяйте, не приводит ли код к дедлокам (например, из‑за захвата контекста без `ConfigureAwait(false)` в библиотечных методах).
4. Рефлексируйте: сравните синхронную и асинхронную реализации задачи — оцените выигрыш в отзывчивости и использовании потоков.
