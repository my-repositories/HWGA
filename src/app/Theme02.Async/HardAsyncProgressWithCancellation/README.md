## AsyncThrottling (Medium)
Задача: Реализовать асинхронный обработчик очереди задач, который ограничивает количество одновременно выполняемых операций (например, не более N запросов к API одновременно).
Пример:
Вход: tasksCount = 10, maxParallelism = 3
Выход: Execution finished
Объяснение: Несмотря на наличие 10 задач, в любой момент времени в консоль должны выводиться сообщения о выполнении не более чем от 3-х потоков.
Ограничения:

* Использовать SemaphoreSlim для ограничения параллелизма;
* Обработка ошибок в одной задаче не должна прерывать выполнение остальных.

Подсказка: Используйте шаблон try-finally для гарантированного освобождения слота семафора. Пространство имён: namespace HWGA.Асинхронность.Medium.AsyncThrottling;.
Что проверить в решении:

* Соблюдение лимита одновременно работающих задач;
* Обработку исключений внутри асинхронного цикла;
* Корректное освобождение ресурсов (Dispose) семафора.

Решение: Смотрите Program.cs.
